# ファイル名を変更して保存

[![Direct](https://img.shields.io/badge/Direct%20Link-Ai--FileNameManager.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/document/Ai-FileNameManager.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

`catalog-v3.ai` を開いて作業して、さて次の版はどうするか。Finder に切り替えて複製して、名前を `catalog-v4.ai` に直して、開き直して……という往復は地味に面倒です。

しかもファイル名は、放っておくと `ｶﾀﾛｸﾞ 最新㈱ (2).ai` のように育ちます。半角カナ、全角スペース、丸数字、絵文字。どれも「その場では困らないが、あとで困る」たぐいのものです。

そこで、開いているドキュメントのファイル名を**構成要素に分解して**、UI から組み立て直せるようにしました。プレビューを見ながら整えて、そのまま保存できます。

<img alt="" src="" width="50%" />

## 使い方

1. ファイル名を変えたいドキュメントを開く
2. スクリプトを実行
3. ダイアログでプレビューを見ながら設定を調整
4. ［OK］

保存先は**常に現在のファイルと同じフォルダ**です。未保存のドキュメントの場合は、ダイアログが開く前に保存先フォルダを訊かれます。

## 動作：3 つの保存方法

| 動作 | 挙動 |
| --- | --- |
| 元ファイルをリネーム | 新しい名前で保存したあと、元ファイルをゴミ箱へ移動 |
| 別名で保存 | 新しい名前で保存。元ファイルは残り、作業中のドキュメントが新ファイルに切り替わる |
| コピー（複製）を保存 | 元ファイルに上書き保存したうえで、別名のコピーを作成。作業中のドキュメントは元ファイルのまま |

初期選択は常に「別名で保存」です。

違いが出るのは**保存後にどのファイルを編集し続けるか**です。「別名で保存」は新ファイル側に移りますが、「コピー（複製）を保存」は元ファイル側に留まります。区切りのスナップショットだけ残して作業は続けたい、という場合は後者になります。

「元ファイルをリネーム」だけは注意が必要です。元ファイルを参照している他のドキュメント（配置した .ai、InDesign のリンクなど）はリンク切れになります。なお削除ではなくゴミ箱への移動なので、取り返しはつきます。ゴミ箱に同名ファイルがあれば連番を付けて退避し、万一「ゴミ箱にコピーできたが元ファイルを消せなかった」場合はコピーのほうを後始末して、ファイルが二重に残らないようにしています。

## モード：全体 / バージョンのみ

| モード | 挙動 |
| --- | --- |
| 全体 | すべての整形を適用 |
| バージョンのみ | 元ファイル名の v 番号だけを繰り上げ、他の整形はしない |

「バージョンのみ」は、`report-final-v03.ai` を `report-final-v04.ai` にするだけ、という用途です。桁数は維持されます。v 番号が無ければ末尾に `-v2` を付けます。

このモードでは「ファイル名の設定」パネルと「構成要素の順序」パネルが丸ごと非表示になります。整形を一切かけないので、設定しても意味がないためです。プリセットの保存もしません。

元ファイルに**タイムスタンプが無く、v 番号がある**場合は「バージョンのみ」が初期選択になります。それ以外は「全体」です。

## ファイル名の設定

ファイル名は base / サブテキスト / ステータス / タイムスタンプ / 連番 / バージョン番号 の 6 つのセグメントに分解され、それぞれを個別に編集できます。

### ベース

ファイル名の先頭部分。元ファイル名から検出した値が初期値として入ります。空欄にすれば省略できます。

**先頭トークンは常に base として扱われます。** つまり `20260712-catalog.ai` の `20260712` はタイムスタンプではなく base です。ここを日付として解釈すると、日付始まりの命名規則を使っている人のファイル名が壊れるためです。

### サブテキスト

| 選択肢 | 値 |
| --- | --- |
| なし | 付けない |
| 指定 | 入力欄の文字列 |
| 親フォルダー | ファイルが入っているフォルダー名 |
| 2 階層上のフォルダー | その 1 つ上のフォルダー名 |

元ファイル名のうち、base・ステータス・日付・連番・バージョン以外のテキスト部分がサブテキストとして検出され、「指定」の入力欄に入ります。検出できなければ「なし」が強制されます。該当するフォルダーが無い場合は「親フォルダー」「2 階層上のフォルダー」が無効になります。

### ステータス

制作ステータスを dropdown から選びます。`wip：作業中・仕掛かり中` のように表示されますが、**ファイル名に入るのは「：」より前の `wip` だけ**です。

用意されているのは wip / draft / review / revised / updated / fixed / approved / rejected / archived、区切り線をはさんで flattened / outlined です。元ファイル名にこれらの語が含まれていれば自動検出されます。

### タイムスタンプ

「なし」「YYYYMMDD」「YYYY-MM-DD」から選択（初期値は YYYYMMDD）。「時刻も付与」で末尾に `-HHMM` が付きます。1 日に複数版を出すときに。

「なし」を選ぶと、元ファイル名に日付があっても削除されます。

日付は**常に今日の日付**です。元の日付を引き継ぐのではありません。

### 連番

`page01` `page001` のような連番セグメントを追加します。プレフィックス（既定は `page`）と桁数（01 / 001）を指定できます。

元ファイル名に `pageNN` があれば自動検出し、プレフィックスと桁数を初期値に反映したうえで、**プリセットより優先して必ずオン**にします。既存の連番が黙って消えるのを防ぐためです。

### バージョン番号

| 選択肢 | 挙動 |
| --- | --- |
| なし | 付けない（元の v 番号があっても削除） |
| v1, v2… | パディング無し |
| v01, v02… | 2 桁ゼロ埋め |
| v001, v002… | 3 桁ゼロ埋め |

初期値は「なし」です。ただし元ファイルに v 番号がある場合だけ、前回の選択（プリセット）が優先されます。

既存の v 番号は +1、無ければ v1 / v01 / v001 から採番します。

## 自動繰り上げ

連番とバージョン番号は、**同じフォルダー内を実際にスキャンして**繰り上げます。

たとえば `report-v03.ai` を開いていて、同じフォルダーに `report-v07.ai` が既にあれば、`report-v04.ai` ではなく `report-v08.ai` になります。走査の対象は「同じ前後（prefix / suffix）を持つ .ai ファイル」なので、無関係なファイルの番号には引きずられません。

計算した番号が現在の番号以下になる場合は繰り上げません。フォルダーの走査に失敗した場合も、既存の番号を下回ることはありません。

## ファイル名の整形

ここから下は「ファイル名として気持ち悪いもの」をまとめて処理する機能です。

| 項目 | 選択肢 | 初期値 |
| --- | --- | --- |
| 区切り記号 | 変更しない / `-` / `_` | `-` |
| 濁点・半濁点の正規化 | 変更しない / 結合する | 結合する |
| クリーンなファイル名 | 削除する / `-` に変更 / `_` に変更 | `-` に変更 |
| 半角カナ → 全角 | チェックボックス | オン |
| 丸数字や法人略記など | 変更しない / 削除する / 変換する | 変換する |

### 区切り記号

`-` `_` `.` を選んだ記号に統一します。`YYYY-MM-DD` 形式のタイムスタンプ（および `-HHMM` を付けたもの）は内部の `-` が保護されるので、`_` に統一しても `2026_07_17` にはなりません。

連続した区切りは 1 つに圧縮され、先頭・末尾の区切りと空白は削除されます。

### 濁点・半濁点の正規化

macOS のファイル名は「が」が「か」＋濁点の 2 文字に分解されていることがあります（NFD）。これを結合済みの 1 文字（NFC）に直します。「あ゙」のような合成できない組み合わせはそのまま残します。

### クリーンなファイル名

OS で使えない文字（`\ / : * ? " < > |`）、絵文字、機種依存文字、スペースをまとめて処理します。「削除する」なら消し、「`-` に変更」なら `-` に置き換えます。

連続するスペースは**先に 1 つへ畳んでから**置換します。そうしないと `-` 選択時に「A  B」が `A--B` になってしまうためです。

判定は許可リスト方式で、ASCII の印字可能文字・CJK 記号・仮名・漢字・ハングル・全角 ASCII・半角カタカナを「標準」とみなし、それ以外を置換対象にしています。

### 半角カナ → 全角

`ｶﾀﾛｸﾞ` を `カタログ` に変換します。`ｶﾞ` `ﾊﾟ` のような濁点・半濁点付きも結合されます。

この機能は**クリーンなファイル名で「`-` に変更」「`_` に変更」を選んでいるときだけ**有効です。「削除する」だと半角カナが消えてしまうため、変換する意味がないからです。

### 丸数字や法人略記など

㈱→株、①→1、Ⅰ→1、℡→TEL、№→No、㎜→mm、～→~、—→- のように ASCII 相当へ変換します。丸数字は白丸（①〜⑳）、黒丸（❶〜❿、⓫〜⓴）、括弧付き（⑴〜⒇）に対応。ダッシュ類も 8 種類が半角ハイフンに寄せられます。

「削除する」を選ぶと、これらの文字は変換せず消えます。

処理の順番は **半角カナ → 変換 → クリーン** です。先にクリーンをかけると ㈱ が「機種依存文字」として消えてしまい、株に変換する機会が失われるためです。

## 構成要素の順序

| 選択肢 | 挙動 |
| --- | --- |
| 標準順 | base → サブテキスト → ステータス → タイムスタンプ → 連番 → バージョン |
| 現在のファイル名に準じる | 元ファイル名で実際に出現した順 |
| カスタム順 | ［編集］ボタンで ↑↓ で並び替え |

初期値は「現在のファイル名に準じる」です（検出できるものが無ければ「標準順」）。ここだけはプリセットを見ずに毎回リセットされます。

「現在のファイル名に準じる」を選んでいるときに構成要素（ベース／サブテキスト／ステータス／タイムスタンプ／連番／バージョン）を変更すると、**自動的に「標準順」へ降格します**。元のファイル名を基準にした並びなのに、その中身を変えてしまえば前提が崩れるためです。整形系（区切り記号やクリーンなど）は並び順に影響しないので、降格しません。

なお「現在のファイル名に準じる」は検出された要素の並びですが、連番のように元ファイル名に無かった要素は正規の位置に補完されます。オンにしたのに出力から漏れる、ということが起きないようにするためです。

## 対象

アクティブな Illustrator ドキュメント（保存形式は .ai）。

## 補足

**ファイル名が長すぎる場合**：拡張子込みの UTF-8 バイト長が 240 バイトを超えると確認ダイアログが出ます。続行はできます。

**Windows 予約名**：`CON` `PRN` `AUX` `NUL` `COM1`〜`COM9` `LPT1`〜`LPT9` になってしまった場合は、末尾に `_` を足して回避します。

**保存先フォルダはダイアログを開く前に確定します。** プレビューに出る繰り上げ後の番号と、実際に保存されるファイル名を一致させるためです。

**前回の設定はプリセットとして保存されます。** 保存先は `Folder.userData` 内の `FileNameManager-prefs.txt` で、`key=value` 形式のテキストです。保存が成功したときだけ書き込まれます。

**スクリプト冒頭のフィーチャースイッチ**：ステータス、並び順、区切り記号、NFC、クリーン、変換、半角カナ、連番の各機能は、スクリプト先頭の `FEATURE_*` を `false` にすると UI ごと消えます。使わない機能でダイアログが混み合うのが嫌な場合に。

## 更新履歴

- v1.0（2026-05-27）初期バージョン
- v1.3.5（2026-07-12）mergeFragmentedText が page/date/status/version など認識済みセグメントをタイトルへ再結合しないよう修正（text 値のみ結合）。既存 pageNN 検出時は prefs より優先して連番を必ず ON に。連番の初期候補を「既存番号 +1」とし、フォルダー検索失敗時も既存番号を下回らないように。「ファイル名の設定」パネルのラベルを右揃えに変更。loadPrefs が CRLF 保存された prefs の行末を除去するよう堅牢化。moveToTrash がゴミ箱へコピー後にオリジナル削除へ失敗した場合、コピーを後始末して二重化を防止。先頭トークンは常に base（先頭日付も base）である仕様とバージョン番号の初期値が「なし」であることをコメントへ明記。内部の変数名・関数名を推測しやすい具体名へ整理（挙動不変）。概要コメントを最新実装に更新。
- v1.3.6（2026-07-23）バージョン／連番の自動繰り上げを、整形（クリーン・変換・区切り記号統一）の**あと**に実行するよう修正。フォルダー内の既存ファイル名は整形済みで保存されているため、整形前の名前で照合すると最大値を取りこぼして繰り上がらないことがあったのを解消（プレビューと実処理の両方）。UI レイアウトの共通設定（`setupWindow` / `setupPanel` などと余白定数）を 1 か所に集約し、メインダイアログと並び順サブダイアログのウィンドウ余白を統一。
