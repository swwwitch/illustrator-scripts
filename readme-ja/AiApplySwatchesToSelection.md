# カラーを配色

[![Direct](https://img.shields.io/badge/Direct%20Link-AiApplySwatchesToSelection.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/color/AiApplySwatchesToSelection.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

複数のオブジェクトやテキストに何色かをぐるぐる回しながら適用したい、という場面があります。ロゴのバリエーション、見出しの1文字ずつの色分け、図形の塗り分けなど。

手作業だと「1つ選んでスウォッチをクリック、次を選んでまた別のスウォッチ」の繰り返しになりますが、そもそも**どの配色がいいのかは並べてみないと分からない**のが厄介なところです。

そこで、配色の単位と順番をラジオボタンで切り替えながら、**その場で結果を見比べられる**常駐パレットにしました。

## 使い方

1. 色を適用したいオブジェクト（またはテキスト）を選択
2. スウォッチパネルで使いたいスウォッチを選択
3. スクリプトを実行

パレットが開いた時点で選択中のスウォッチを取り込み、■のチップとして表示します。あとはラジオボタンを切り替えるたびにライブプレビューされるので、気に入った状態でパレットを閉じれば確定です。

**適用ボタンはありません。** 閉じた時点の結果がそのまま残ります。やっぱりやめる場合は `Cmd + Z` で戻してください。パレットは `Esc` キーでも閉じられます。

### 配色単位

| 単位 | 動作 |
| --- | --- |
| オブジェクト | 選択オブジェクトごとに1色 |
| 1文字 | テキストの1文字ずつに1色 |
| 単語 | 単語ごとに1色（欧文向け） |
| 行 | 行ごとに1色 |
| 段落 | 段落ごとに1色 |

選択内容に合わない単位は自動的にディム（グレーアウト）します。図形だけを選んでいるときは文字/単語/行/段落が、オブジェクトが1つだけのときは「オブジェクト」が選べません。パレットを開いたまま選択を変えても、ディムは追従して更新されます。

「単語」は、各行の先頭色が互い違いになるように配色します。行頭がいつも同じ色から始まると、いかにも機械的な仕上がりになるためです。句読点やスペースは直前の単語と同じ色になります。

### 配色順

| 順番 | 動作 |
| --- | --- |
| 取り込み順 | チップの並び順のまま適用 |
| 逆順 | チップの並びを逆にして適用 |
| ランダム | 並びをシャッフルして適用 |
| 完全ランダム | 適用先ごとに毎回抽選 |

「ランダム」と「完全ランダム」の違いが分かりにくいかもしれません。

「ランダム」は**並び順をシャッフルするだけ**です。3色を10個のオブジェクトに適用するなら、シャッフルされた3色が3回ちょっと繰り返されます。ランダムではあるものの、周期的なパターンは残ります。

「完全ランダム」は**適用先ごとに毎回サイコロを振ります**。繰り返しパターンが出ない代わりに、同じ色が隣り合うこともあります。規則性を消したいときはこちら。

なお「完全ランダム」でも、単語単位のときは単語ごとに1色です（単語の中で文字がバラバラの色になったりはしません）。

## スウォッチを選んでいないときは

自動的にカラーを生成します。

- CMYK ドキュメント：CM / CY / MY の2色掛け合わせ（K=0）から、なるべく似た色にならないよう生成
- RGB ドキュメント：あらかじめ用意した6色

## 2つの読み込みボタン

- **［スウォッチを読込］**：スウォッチパネルの選択を取り込み直します
- **［塗り色を読込］**：選択オブジェクトの塗り色を取り込み直します

「塗り色を読込」は、すでにいい感じに色がついているオブジェクトから配色だけを拝借して、別の並び順で配り直したいときに使います。

取り込んだ配色はスウォッチ名（または色の値）で保持しているため、パレットのラジオボタンをクリックしてスウォッチパネルの選択が外れても影響を受けません。

## 対象

TextFrame、PathItem、CompoundPathItem（グループ内も再帰的に処理）

## 補足

Illustrator の常駐パレットは、表示している間に DOM への接続を失うという事情があるため、実際のオブジェクト操作はすべて BridgeTalk 経由でメインエンジンに委譲しています。

プレビューの取り消しには `app.undo()` を使わず、書き換える前の塗り・線・不透明度をスナップショットとして保存し、そこから復元する方式にしています。`app.undo()` はドキュメント全体の履歴を巻き戻してしまうため、パレットを開いたまま手作業で編集したぶんまで消えてしまうためです。
