# 段落を選択

[![Direct](https://img.shields.io/badge/Direct%20Link-AiSelectParagraph.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/text/AiSelectParagraph.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

段落まるごとにスタイルを当てたい、コピーしたい、消したい。そんなとき、テキストの先頭からドラッグして末尾でボタンを離す、という操作を毎回しています。長い段落だと画面をスクロールしながらのドラッグになりますし、勢い余って次の段落まで巻き込んでしまうこともあります。

InDesign なら「段落を選択」に相当する操作がありますが、Illustrator にはありません。それならスクリプトで、と作ったのがこれです。**カーソルを置いて実行するだけ**で、その段落の先頭から末尾までが選択されます。

<img alt="" src="" width="50%" />

## 使い方

1. テキストにカーソルを置く（文字ツールで編集中の状態）
2. スクリプトを実行

それだけです。ダイアログもパレットもありません。同じ段落内で文字列を選択した状態で実行した場合も、その段落全体に広がります。

オブジェクトを選択しただけの状態では実行できません。テキスト編集中のときだけ `app.selection` が TextRange になるため、そこを動作条件にしています。

## Illustrator の「段落」の落とし穴

このスクリプトの読みどころはここです。

Illustrator の `paragraphs` コレクションは、**強制改行（Shift + Return、`\u0003`）でも段落を区切ってしまいます**。見た目には「1つの段落を途中で折り返しただけ」でも、DOM 上は別々の段落として数えられるということです。

そのため `paragraphs` は使わず、**ストーリー全文を `\r`（改行）で走査して段落の境界を求めています**。強制改行は段落内の折り返しとして扱うので、そこで選択が止まることはありません。「見た目どおりの段落」が選択されます。

なお全文の取得には `story.textRange.contents` を使っています（`story.contents` というプロパティは存在しません）。

## 段落末尾の改行を含めるか

スクリプト冒頭の `INCLUDE_PARAGRAPH_RETURN` で切り替えられます（既定は `true`）。

| 値 | 動作 |
| --- | --- |
| `true` | 段落末尾の改行まで選択。改行だけの空段落も選択できる |
| `false` | 改行の手前まで選択。空段落は選択できない |

`false` のときに空段落でカーソルを置いて実行すると、選択できる文字が無いため中止します。

強制改行は改行として扱わないため、この設定の影響を受けません。

## 中止するケース

| 状況 | メッセージ |
| --- | --- |
| ドキュメントが開かれていない | ドキュメントを開いてください。 |
| テキストにカーソルが無い | テキストにカーソルを置いてください。 |
| 選択が段落をまたいでいる | 段落をまたいでいます。1つの段落内にカーソルを置いてください。 |
| 空段落（`INCLUDE_PARAGRAPH_RETURN` が `false`） | この段落には選択できる文字がありません。 |

段落をまたぐ選択は、どの段落を対象にすべきか決められないので中止します。

## 対象

テキスト編集中の TextRange（そのカーソルが属するストーリー）

## 補足

`TextRange.end` は排他的です。つまり段落末尾の改行まで選択した状態では、`end` はすでに次の段落の先頭を指しています。これを素直に扱うと、

- またぎ判定：1段落しか選んでいないのに「段落をまたいでいる」と誤判定する
- 段落末尾の探索：選択済みの改行を飛び越えて、次の段落の改行を拾ってしまう

という2つの誤りが起きます。

そこで、またぎ判定は `end - 1`（選択がある場合のみ）で行い、段落末尾の改行検索は選択の終了位置ではなく**段落の先頭を起点**にしています。どちらも v1.0.1 で修正した箇所です。
