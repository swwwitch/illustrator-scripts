# パスファインダー

[![Direct](https://img.shields.io/badge/Direct%20Link-AiSmartPathfinder.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/fx/AiSmartPathfinder.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

Illustrator のパスファインダーパネルは、上段のアイコンをそのままクリックすると複合シェイプになり、Option を押しながらクリックすると実際のパスになります。この「押し方で結果が変わる」仕様は、慣れていても取り違えます。

しかも、実際にやりたいことはパスファインダーだけでは終わりません。マドを埋めたい、線を塗りに変換したい、ライブ効果として乗せたアピアランスを分割したい、うっかり残った効果だけを消したい……。そのたびにパネルとメニューを行き来することになります。

そこで、**出力の形をラジオボタンで先に決めてから、アイコンをクリックするだけ**で済む常駐パレットにしました。周辺の操作もまとめて同じパレットに載せています。

## 使い方

1. オブジェクトを選択
2. スクリプトを実行
3. 「モード」で出力の形を選び、アイコンをクリック

クリックした時点で即実行されます。適用ボタンはありません。パレットは `Esc` キーで閉じられます。

パレットは「基本」「その他」の2タブ構成です。

## モード（出力の形）

「基本」タブのいちばん上で、**結果をどの形で残すか**を先に決めます。既定は「パスに変換」です。

| モード | 動作 | ショートカット |
| --- | --- | --- |
| パスに変換 | グループ化 → ライブ効果を適用 → 拡張 → グループ解除。実際のパスになる | `P` |
| 複合シェイプを作成 | 複合シェイプのまま残す（形状モードの4つだけ） | `C` |
| 効果として適用 | ライブ効果のまま乗せる（拡張しない） | `F` |

「複合シェイプを作成」を選んでいる間は、**下段のパスファインダー6つがディム（無効）になります**。複合シェイプは形状モード専用のためです。

### 形状モード（上段）

| アイコン | 操作 |
| --- | --- |
| 合体 | Unite (Add) |
| 前面オブジェクトで型抜き | Minus Front |
| 交差 | Intersect |
| 中マド | Exclude |

### パスファインダー（下段）

| アイコン | 操作 |
| --- | --- |
| 分割 | Divide |
| 刈り込み | Trim |
| 合流 | Merge |
| 切り抜き | Crop |
| アウトライン | Outline |
| 背面オブジェクトで型抜き | Minus Back |

### Option+クリックで一時的に上書き

モードを切り替えずに、その1回だけ別の形で出したいことがあります。

- **形状モード（上段）を Option+クリック** → モードに関係なく複合シェイプを作成
- **パスファインダー（下段）を Option+クリック** → モードに関係なく効果として適用

パスファインダーパネルの Option+クリックとは逆向き（パネルは Option でパス化、こちらは Option で複合シェイプ／効果）ですが、「Option を押すと拡張しない側に倒れる」と覚えると分かりやすいと思います。

## オプション

| オプション | 既定 | 動作 |
| --- | --- | --- |
| 余分なポイントを削除 | ON | ライブ効果の `RemovePoints` |
| ［強制］ | — | 選択パスの直線上の冗長なアンカーポイントを削除（許容誤差 0.02 固定） |
| 塗りのないアートワークを削除 | OFF | ライブ効果の `ExtractUnpainted` |
| ［複合シェイプを拡張］ | — | 選択中の複合シェイプを通常のパスへ拡張 |

「塗りのないアートワークを削除」は**分割・アウトラインのときだけ**、しかも**「パスに変換」モードのときだけ**効きます。ライブ効果のまま `ExtractUnpainted` を有効にすると分割・アウトラインの見た目が壊れるため、「効果として適用」のときは強制的に OFF になります（Option+クリックで効果として適用したときも同様です）。

［複合シェイプを拡張］は **Option+クリックで「解除」に変わります**。Option を押している間はボタンのラベル自体が「複合シェイプを解除」に切り替わるので、押す前に確認できます。

なお、このボタンは常に押せます。パレット側では選択の変更を受け取れないため、複合シェイプがあるかどうかはクリックした時点で判定する作りになっています。

## 「その他」タブ

| パネル | ボタン | 動作 |
| --- | --- | --- |
| マド埋め | パスに変換 | 複合パス解除 → ライブパスファインダー合体 → 拡張 → グループ解除 |
| マド埋め | 効果として適用 | 複合パス解除 → ライブパスファインダー合体（ライブ効果のまま） |
| 変換 | 線を塗りに変換 | 線をアウトライン化して1つの塗りにまとめる（ライブ効果） |
| アピアランス | 分割 | アピアランスを実体化し、可能ならグループ解除 |
| アピアランス | 効果のみを消去 | アピアランスを消去したうえで、元の塗り・線を復元 |
| アピアランス | （完全に）消去 | アピアランスを消去（復元なし） |
| ツール、パネルを表示 | 「アピアランス」パネル／「パスファインダー」パネル | 該当パネルを表示 |
| ツール、パネルを表示 | シェイプ形成ツール／選択ツール | 該当ツールに切り替え |

### 「効果のみを消去」と「（完全に）消去」の違い

名前が似ていますが、結果はかなり違います。

「（完全に）消去」は、アピアランスパネルのメニューにある「アピアランスを消去」そのままです。ライブ効果も塗り・線もまとめて消えます。

「効果のみを消去」は、いったん同じ「アピアランスを消去」を対象ごとに実行したうえで、**元の塗り・線を復元します**。テキストの場合は文字の塗りを戻します。結果として、見た目上はライブ効果だけが消えたように見えます。

## 選択数の条件

モードによって、必要な選択数が違います。

- **効果として適用**：1つ以上
- **パスに変換／複合シェイプを作成**：2つ以上

ただしこの「2つ以上」は、**グループを解除したあとの数**で判定します。「パスに変換」「複合シェイプを作成」は実際のパスを対象にするため、選択にグループが含まれていれば実行前に `ungroupAll` で解除するからです。そのため、**グループを1つだけ選んでいる状態でも、中身が2つ以上あれば実行できます**。

「効果として適用」は効果をグループごと乗せるので、解除しません。

## 取り消しは2回必要なことがあります

「パスに変換」「複合シェイプを作成」でグループの解除が発生した場合、**解除は独立した undo 履歴になります**。つまり元に戻すには `Cmd + Z` を2回押す必要があります。

## 対象

選択中のオブジェクト（グループ・複合パスを含む）。

## 補足

Illustrator の常駐パレットは、表示している間に DOM への接続を失うという事情があるため、実際のオブジェクト操作はすべて BridgeTalk 経由でメインエンジンに委譲しています。

複合シェイプの作成は、API から直接叩けないためダイナミックアクション（`ai_compound_shape`）を一時的な `.aia` として書き出して再生しています。［複合シェイプを拡張］（`ai_expand_compound_shape`）／［解除］（`ai_release_compound_shape`）、「（完全に）消去」（`ai_plugin_appearance`）も同じ方式です。

いっぽう「パスに変換」モードの形状モードは、ダイナミックアクションで作った複合シェイプが `expandStyle` で焼き込めないため、複合シェイプを経由せず Adobe Pathfinder のライブ効果 XML を適用してから拡張しています。

アイコンは `onDraw` で描画しており、UI の明暗（`uiBrightness`）に応じて色が切り替わります。無効時は背景色に寄せてディム表示されます。

Option の押下状態は `onClick` からは読めないため、直前の `mousedown` で記録した値を読む方式にしています。

二重起動はしません。すでに開いていれば、作り直さずに前面化します。
