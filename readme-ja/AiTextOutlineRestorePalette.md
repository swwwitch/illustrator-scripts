# テキストのアウトライン化と復元

[![Direct](https://img.shields.io/badge/Direct%20Link-AiTextOutlineRestorePalette.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/outline/AiTextOutlineRestorePalette.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

入稿前にテキストをアウトライン化したあとで、「やっぱりこの1文字だけ直したい」という話になることがあります。

アウトライン化してしまうと、もうそこにあるのはパスの集まりです。何のフォントで、何級で、行送りはいくつだったのか。カーニングは？　禁則は？　元データを開き直せばいいのですが、その元データが手元にないこともあります。

そこで、**アウトライン化する直前の属性をオブジェクトのメモ（note）に書き込んでおき、あとからそれを読んでテキストに戻す**常駐パレットにしました。アウトライン化したファイルさえあれば、そこから復元できます。

## 使い方

### アウトライン化するとき

1. テキストを選択
2. パレットの［アウトライン化（メモ付き）］をクリック

通常のアウトライン化と同じ結果になりますが、できあがったオブジェクトの「メモ」に属性が保存されている点が違います。保存された内容はそのままパレットの一覧に表示されます。

### 復元するとき

1. メモ付きのアウトライン（パスまたはグループ）を選択
2. ［テキストを復元］をクリック

メモを解析して、テキストフレームを作り直します。

パレットは `Esc` キーでも閉じられます。

## 復元されるプロパティ

| 項目 | 備考 |
| --- | --- |
| 文字列 | 複数行に対応 |
| 組み方向 | 縦組み／横組み |
| フォント | PostScript 名で保存 |
| フォントサイズ | |
| 行送り | |
| 自動行送り | ON のときは行送り値を書かず自動計算に戻す |
| 水平比率・垂直比率 | 長体／平体 |
| カーニング | メトリクス／オプティカル／和文等幅／なし |
| プロポーショナルメトリクス | カーニング方式に連動 |
| トラッキング | |
| 文字ツメ | |
| 行揃え | 左／中央／右／均等配置（最終行の扱いも含む） |
| 禁則 | セット名で保存 |
| 文字組みアキ量設定 | セット名で保存 |
| 文字カラー | CMYK／RGB／グレー／スポット |
| 座標 | geometricBounds 基準。一覧には表示されません |

## 復元できないもの・される条件

ここが分かりにくいところなので、まとめておきます。

**属性は「先頭の1文字」の値だけを保存します。** 1つのテキストフレームの中で色やサイズが混在していても、全体が先頭文字の値で復元されます。文字ごとに違う属性を持たせていたものは戻りません。

**グラデーション・パターン塗りは非対応です。** 保存時に空になり、復元後は既定色のままになります（不可視にならないようにこうしています）。

**禁則の「なし」は復元をスキップします。** Illustrator のスクリプトから禁則に「なし」を代入することができないためです。「なし」だった段落は、復元後は既定の禁則のままになります。文字組みアキ量設定のほうは「なし」も復元できます。

**フォントが見つからないときは既定フォントになります。** その場合はステータスに「一部フォントは既定値を使用」と表示されます。他の属性の復元は止まらず、そのまま続行されます。

**英語環境では和文専用の属性を扱いません。** 組み方向・禁則・文字組みアキ量設定・文字ツメの4つは、保存も表示も復元もされません。

**古いメモは、記録のない項目だけをスキップします。** 以前のバージョンで作ったアウトラインでも、記録されている範囲で復元できます。

## 復元オプション

| オプション | ON（既定） | OFF |
| --- | --- | --- |
| アウトラインデータを残す | 元のアウトラインを `outlined_text` レイヤーへ退避 | 復元後にアウトラインを削除。`outlined_text` レイヤーも作らない |
| 復元したテキストを別レイヤーに | `restored_text` レイヤーへ復元 | アウトラインと同じレイヤーへ復元。`restored_text` レイヤーも作らない |

退避された元のアウトラインは、不透明度 30% に落とされ、ロックされます。レイヤー自体も最背面へ送られ、テンプレートレイヤー属性が付いてロックされます。復元したテキストと重なって見えづらくならないようにするためです。

`outlined_text` レイヤーはアウトライン化のたびに増やさず、既存のものがあればロックを解除して再利用します。アウトラインは常にこの1枚に集約されます。

## 2つのボタン

- **［メモを読み込み］**：選択オブジェクトのメモを読み込んで一覧に表示します
- **［属性パネル］**：Illustrator の属性パネルを開閉します（メモの中身を直接確認・編集したいとき用）

パレットを開いた時点でも、選択オブジェクトのメモは自動で読み込まれます。複数選択しているときは、**メモを持っている最初のオブジェクト**の内容を表示します。

## 対象

- アウトライン化：TextFrame
- 復元：PathItem、GroupItem（メモを持つもの）

## 補足

メモは互換性のため日本語のラベルで保存されます。英語環境では、表示の直前に項目名と列挙値だけを英語へ変換しています。数値・フォント名・カラー値はそのまま素通しです。

文字組みアキ量設定の内部名は環境によって表記が揺れる（空白・区切り・大小）ため、名前を正規化してから突き合わせています。それでも引けない場合は、内部のローマ字名を前方一致で解決するフォールバックを持っています。

Illustrator の常駐パレットは表示している間に DOM への接続を失うという事情があるため、実際のオブジェクト操作はすべて BridgeTalk 経由でメインエンジンに委譲しています。

テンプレートレイヤー属性の付与だけは API から直接指定できないため、一時アクションを読み込んで再生しています。アクションはパレットを閉じるときにまとめて解放されます。
