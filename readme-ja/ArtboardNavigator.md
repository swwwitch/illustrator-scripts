# アートボードナビゲーター

[![Direct](https://img.shields.io/badge/Direct%20Link-ArtboardNavigator.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/artboard/ArtboardNavigator.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

アートボードがたくさんあるドキュメントで、「3番の次は5番を見て、やっぱり1番に戻って……」と行き来していると、**いま自分がどこにいるのか分からなくなる**ことがあります。

アートボードパネルをダブルクリックすれば移動はできますが、画面が一瞬でパッと切り替わるため、隣に飛んだのか遠くへ飛んだのかが体感として残りません。前後関係が頭に入らないまま、ひたすら番号を探すことになります。

そこで、**移動の途中経過をアニメーションで見せる**ナビゲーターを作りました。ズームしながらスーッと隣のアートボードへ寄っていくので、位置関係がそのまま印象として残ります。

## 使い方

アートボードが2つ以上あるドキュメントで実行すると、パレットが開きます。1つしかない（または開いていない）ときは、その旨を伝えて終了します。

パレット上部に5つのボタンが並びます。

| ボタン | 動作 |
| --- | --- |
| `\|<` | 最初のアートボードへ（先頭にいるときは無効・グレーアウト） |
| `<` | 前のアートボードへ（端まで来たら循環） |
| `■■` | 全アートボードを画面に収めて表示 |
| `>` | 次のアートボードへ（端まで来たら循環） |
| `>\|` | 最後のアートボードへ（最終にいるときは無効・グレーアウト） |

`<` と `>` は端で循環するので常に有効ですが、`|<` と `>|` は端にいるときだけグレーアウトします。行き止まりが視覚的に分かるようにするためです。

パレットにフォーカスがあるときは、キーボードでも操作できます。

| キー | 動作 |
| --- | --- |
| `Control + Shift + Option + ←` | 前のアートボードへ |
| `Control + Shift + Option + →` | 次のアートボードへ |
| `Control + Shift + Option + ↑` | 全アートボードを一覧表示 |

## オプション

| 項目 | 動作 |
| --- | --- |
| アニメーション | OFF で補間せず瞬時に切り替え（以降の項目はディム表示） |
| スピード | 移動アニメーションの速さ（1〜10、右ほど速い） |
| イーズアウト | ON で加減速あり、OFF で等速移動 |
| Preziライクモード | 前後移動の中盤で一度ズームを引いてから寄せる |
| 俯瞰の強さ | Preziライクモードで引くズーム量（0〜1） |
| アートボード名を表示 | 移動先の左上に「番号：アートボード名」を表示 |

これらの設定と、パレットの表示位置は、次回起動時に引き継がれます。

### スピードと「イーズアウト」

「スピード」は、アニメーションのステップ数を変えています。最も遅い設定で40ステップ、最も速い設定で6ステップです。ステップが少ないほど、短いコマ数で目的地に着くので速く見えます。

「イーズアウト」は easeOutQuad、つまり**出だしが速く、終わりに向かって減速する**カーブです。押した瞬間に動き出すので、開始のタイムラグを感じにくくなります。OFF にすると等速（線形）になります。

なお、ステップ数は設定値そのままではなく、**移動量に応じて自動的に減らされます**。画面に対するパン量とズーム変化率の大きい方を見て、隣へちょっと動くだけなら少ないステップで済ませる、という調整です（下限はスピード設定の20%、最低2ステップ）。

### Preziライクモードと「俯瞰の強さ」

「Preziライクモード」は、A から B へまっすぐ移動するのではなく、**途中で一度ズームを引いて全体を見せてから、目的地に寄っていく**動きです。プレゼンツールの Prezi のような挙動なので、この名前にしています。

引き具合は「俯瞰の強さ」スライダー（0〜1）で決まります。0 なら引かず、1 に近づくほど大きく引きます。引き量は sin カーブで出入りするため、移動の中盤で最も引き、始点と終点では元のズームに戻ります。

このモードが効くのは**個別のアートボードへ移動するときだけ**です。`■■`（全体表示）はそもそも引く動きなので、Prezi の引きは使いません。

## アートボード一覧

「アートボード一覧」チェックボックスで、番号とアートボード名の一覧を表示できます。行をクリックすると、そのアートボードへ移動します。

チェックを外すと、一覧を隠すだけでなく**パレット自体がその高さぶん縮みます**（領域を確保したまま空白にはなりません）。再表示すると、隠す前の高さへ正確に戻ります。手動でリサイズした高さもそのまま保たれます。

パレットはリサイズ可能で、一覧は縦横とも追従して広がります。

## アートボード名のラベル

「アートボード名を表示」を ON にすると、個別のアートボードへ移動した直後に、その左上へ「番号：アートボード名」のラベルが描画されます。

- テキストは白・HiraginoSans-W6、背景は黒の長方形
- 文字サイズはアートボード幅に比例（既定は幅の 3%）するため、画面にフィットしたときの見た目がほぼ一定になります
- 文字サイズ比率・背景／文字の不透明度は、スクリプト先頭の変数（`LABEL_FONT_RATIO` / `LABEL_BACKGROUND_OPACITY` / `LABEL_TEXT_OPACITY`）で調整できます

ラベルは `ArtboardNavigator` という**専用レイヤー**（ロック ON・プリント OFF）に描かれ、切り替えのたびに作り直されます。チェックを OFF にしたとき、全体表示に切り替えたとき、パレットを閉じたときには、このレイヤーごと削除されます。作業レイヤーを巻き込まないよう、描画後にアクティブレイヤーは元へ戻されます。

## 対象

アートボード（ドキュメント内のアートボードのビュー移動のみ。既存のオブジェクトには手を加えません）。

## 補足

目標のズーム値は、`view.bounds` と現在のズームから**数式で算出**しています。`fitin` / `fitall` などのメニューコマンドを実行して測るとその時点で画面が動いてしまい、アニメーション開始前にタイムラグが出るためです（`view.bounds` を取得できない環境向けに、メニューで実測して元へ戻すフォールバックは用意しています）。

Illustrator の常駐パレットは表示している間に DOM への接続を失うため、実際のビュー操作とアニメーションは BridgeTalk 経由でメインエンジンへ送っています。しかも、毎回すべてのコードを送るのではなく、**起動直後に一度だけ本体をメインエンジンへ登録**しておき、以降のクリックでは短い呼び出し文だけを送ります。送信ペイロードが最小になるので、クリックからの反応が軽くなります。

一覧の選択は、BridgeTalk の応答（`onResult`）を待たずにパレット側で確定させています。移動先の絶対インデックスをパレット側で先に決め、その番号をワーカーへ渡すことで、一覧の選択と実際の移動先が食い違わないようにしています。

パレット位置は、モニタ構成が変わって画面外になっていた場合は復元せず、既定位置で開きます。

ボタンの配色は、環境設定「ユーザーインターフェイスの明るさ」に追従します（明るい UI では白バック＋枠線＋黒記号、暗い UI では黒バック＋白記号）。パレットをアクティブにしたタイミングで明るさを見直すので、環境設定を変えた場合もそのまま反映されます。
