# ポイント文字・パス上文字とエリア内文字の相互変換

[![Direct](https://img.shields.io/badge/Direct%20Link-ConvertAreaAndPointType.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/text/ConvertAreaAndPointType.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

### 概要

- 選択内容に応じて、ポイント文字・パス上文字 → エリア内文字（順変換）、エリア内文字 → ポイント文字（逆変換）を自動で切り替えて実行するスクリプト。
- 順変換では見た目を保ったままエリア内文字化し、逆変換では元の枠サイズを保ったままポイント文字化できる。
- ダイアログは順／逆共通（タイトル「テキストの変換」）で、現在の選択の内訳とスタイルの保持有無を表示する。

### 主な機能

- **選択オブジェクト**：現在の選択を「ポイント文字 ×2」のように種類別に集計して表示
- **スタイル：保持する**（初期値）／**保持しない**：順変換・逆変換で意味が異なり、それぞれツールチップで補足
  - 順変換・保持する：元テキストの見た目（塗り・線・効果）を一時グラフィックスタイルとして登録し、変換後に適用（適用後に一時スタイルは削除）
  - 順変換・保持しない：グラフィックスタイルを適用せず、文字だけを移す
  - 逆変換・保持する：変換後の見た目のまま、追加処理なし
  - 逆変換・保持しない：塗りを2枚追加し、元の枠サイズ（幅A×高さB）の固定サイズ［形状に変換：長方形］効果でボタン風の背景を付与
- フレームサイズは、複製→アピアランス分割→アウトラインで計測した実寸を基準に決定
- 順変換では行揃え・テキストの配置とも中央揃え。自動カーニングと文字組みアキ量設定も引き継ぐ
- パス上文字は内部で一度ポイント文字に分離（文字ごとの属性・行揃えを保持）してから同じ経路で処理
- 縦中央配置とグラフィックスタイル登録には一時的なダイナミックアクションを使用し、終了時に自動で破棄するためアクションパネルに残骸が残らない
- 1件も変換できなかった場合は無言で終わらず、理由を添えて警告を表示
- 日本語／英語の自動切り替え

### 処理の流れ

1. 選択にエリア内文字が含まれていれば逆変換、含まれていなければ順変換と判定
2. 共通ダイアログで選択の内訳を確認し、スタイルの保持有無を選択（キャンセルで中止）
3. 順変換：計測した実寸の長方形を作成 → エリア内文字化 → 内容・フォント・カーニング・文字組みを移して中央配置（保持する場合は一時グラフィックスタイルを適用）
4. 逆変換：元の枠サイズを取得 → ポイント文字化（一時マーカー名で変換後の参照を確実に回収）→ 保持しない場合は長方形背景を付与
5. 変換結果を選択状態にして終了

### 対象外

- ドキュメントが開かれていない場合
- 何も選択されていない場合、および選択にテキストが含まれていない場合
- テキスト以外のオブジェクト（選択の内訳では「テキスト以外」として集計されるが変換されない）
- 複数選択時はそれぞれ個別に変換される
