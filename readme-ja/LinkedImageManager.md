# リンク画像の管理

[![Direct](https://img.shields.io/badge/Direct%20Link-LinkedImageManager.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/link/LinkedImageManager.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

入稿前のチェックで、リンク画像まわりを一通り確認する場面があります。リンクは切れていないか、解像度は足りているか、RGB のままの画像が混ざっていないか、同じ画像を何箇所で使っているのか。

Illustrator の［リンク］パネルでも1つずつ選べば分かるのですが、**一覧で見比べられない**のが厄介なところです。PPI を確認したければ画像を1つ選んでパネルを展開し、次の画像を選んでまた展開し、の繰り返しになります。「PPI が低い順に並べたい」といった見方はできません。

そこで、配置画像を全部まとめて解析して**表として並べ、ソートと絞り込みができる**常駐パレットにしました。一覧から再リンクやリネーム、削除まで済ませられます。

## 使い方

スクリプトを実行するとパレットが開き、ドキュメント内の配置画像（PlacedItem）が一覧表示されます。

実行前に配置画像を1つ選択していた場合は、対応する行が最初から選択された状態で開きます。このときカンバスのビューは動きません。

ドキュメントを開いていない状態でも、パレット自体は開きます。あとからドキュメントを開いて［更新］ボタンを押せば読み込まれます。配置画像が0件のときやドキュメント未オープンのときは、オプション類とリストを隠した簡易表示になります。

パレットは `Esc` キーでも閉じられます。

## 一覧の見かた

先頭のアイコン列とファイル名列は常に表示され、それ以外の列は［表示列］パネルのチェックボックスで出し入れします。

| 列 | 内容 |
| --- | --- |
| アイコン | ✓ リンク正常 / ⚠ リンク切れ / ⟳ 更新が必要 |
| ファイル名 | リンクファイル名 |
| サイズ | ファイルサイズ |
| 使用数 | 同じファイルが何箇所に配置されているか |
| 幅(mm) / 高さ(mm) | 配置されている実寸 |
| スケール | 配置時の拡大縮小率 |
| PPI | 実効解像度 |
| カラースペース | カラーモード（＋ICC プロファイル名） |
| アートボード | 何番のアートボードに乗っているか |

### ステータスの判定

「⟳ 更新が必要」は、ドキュメントの XMP メタデータに記録されているリンク時の更新日時と、実ファイルの現在の更新日時を比べて判定しています。**5秒を超える差**があれば「更新が必要」です。ファイルが存在しなければ「⚠ リンク切れ」になります。

### PPI とスケール

PPI は、リンクファイルの実ピクセル数と配置寸法から `ピクセル数 × 72 ÷ 配置寸法(pt)` で算出し、縦横の平均を四捨五入した値です。［リンク］パネルの表示と同じ考え方ですが、一覧で並べてソートできます。

ピクセル数の取得には、リンクファイルのバイナリを直接読んでいます。対応しているのは **PNG / JPEG / PSD** です。それ以外の形式（AI、PDF、TIFF など）では PPI とカラースペースが `---` になります。

スケールは配置アイテムの変形マトリックスから求めています。縦横の倍率が 0.1% 以上違うときだけ `80.0% × 60.0%` のように両方を表示し、揃っていれば1つだけ表示します。

### カラースペース

リンクファイルから読み取ったカラーモードと、埋め込まれた ICC プロファイル名を `RGB（sRGB IEC61966-2.1）` のような形で表示します。プロファイルが無ければカラーモードだけです。こちらも PNG / JPEG / PSD が対象です。

## 同一ファイルをまとめる

初期状態では **同じリンクファイルを1行にまとめて** 表示します。ファイルパスが一致するものが同一ファイルの扱いです。「使用数」列にその件数が出て、「アートボード」列には配置されているアートボード番号がまとめて（`1, 3, 5` のように）並びます。

チェックを外すと、配置ごとに1行ずつの表示に切り替わります。

なお**リンク切れの画像はまとめられません**。パスが取れないため、同じ名前に見えても別々の行として出ます。

初期のソート順は「使用数」の降順です。使い回している画像から目に入るようにしています。

## ソート

［ソート］パネルの並び順ドロップダウンと、昇順／降順のラジオボタンで並び替えます。

| 並び順 | 備考 |
| --- | --- |
| ファイル名 | |
| サイズ | 「サイズ」列の表示中のみ |
| 使用数 | 「使用数」列の表示中のみ |
| アートボード | アートボードで絞り込んでいないときのみ |
| 幅 / 高さ / スケール / PPI | 「サイズ、%、PPI」列の表示中のみ |
| ステータス | リンク切れ → 更新が必要 → 正常 の順 |
| カラースペース | 「カラースペース」列の表示中のみ |

**ドロップダウンの項目は表示列に連動して増減します。** 表示していない列では並べ替えられないためです。列を出し入れしても、選んでいた並び順は可能な限り維持されます。

サイズ・使用数・幅・高さ・スケール・PPI を選ぶと、**自動的に降順に切り替わります**。これらは「大きいもの」「多いもの」から見たいことがほとんどだからです。ファイル名やステータスを選んだときは切り替わりません。

値が取れなかった行（PPI が `---` など）は、昇順・降順どちらでも常に末尾にまとまります。

## 絞り込み

### ステータス

「✓ リンク正常」「⚠ リンク切れ」「⟳ 更新が必要」の3つのチェックボックスで、表示するステータスを選びます。リンク切れだけを見たいときは他の2つを外します。

### アートボード

ドロップダウンでアートボードを選ぶと、そのアートボード上の画像だけに絞り込まれます。

**画像が1枚も乗っていないアートボードはディム（グレーアウト）されて選べません。** どのアートボードを見れば良いかが、開いた時点で分かるようにしています。

アートボードを選ぶと、**カンバスのビューもそのアートボードにフィット**します。一覧の絞り込みと画面の表示が連動する形です。

隣の ◀ ▶ ボタンでアートボードを送れます。ディムされているアートボードは自動的に読み飛ばし、末尾まで行くと先頭に戻ります（「すべて」には戻りません）。選べるアートボードが1つ以下のときはボタンごと無効になります。

なお**アートボードで絞り込んでいる間は「アートボード」列が消えます**。全部同じ番号になり、意味が無いためです。

画像がどのアートボードに属するかは、画像の可視バウンズの中心座標がどのアートボードの矩形に入るかで判定しています。どこにも入らなければ `-` です。

## 選択との連動

行を選ぶと、対応する配置画像がカンバス上でも選択され、画面いっぱいにズーム・センタリングされます。同一ファイルをまとめている行なら、その全ての配置がまとめて選択されます。

ズームが不要なら［選択時にズーム表示］のチェックを外してください。

## ファイルパスの表示

リストで選んだ行のパスが［ファイルパス］パネルに出ます。3つのチェックボックスで表示を切り替えます。

| チェック | 動作 |
| --- | --- |
| フルパス | 絶対パスをそのまま表示 |
| Dropboxパスを短縮 | Dropbox のマウントパスの接頭辞を省く |
| ファイル名 | ON でファイル名まで、OFF ならフォルダーまで |

初期状態では「フルパス」「ファイル名」がオフ、つまり**ホームディレクトリを `~` に置き換えたフォルダーパス**が表示されます。パスが長くなりがちなので、既定では短い表示にしています。

「Dropboxパスを短縮」を ON にしている間は「フルパス」がディムになります（短縮とフルパスは両立しないため）。

Dropbox の短縮は、スクリプト冒頭の `DROPBOX_PREFIX` に自分のローカルマウントパスを設定して使います。`""` にしておくと**チェックボックス自体が表示されません**。

## リンクフォルダー一覧

配置画像が参照しているフォルダーが、重複を除いて一覧されます。リストで行を選ぶと、そのファイルが入っているフォルダーが自動的にハイライトされます。

ダブルクリック、または［開く］ボタンで Finder で開きます。

## 画像ごとの操作

リストで行を選んでいるときに使えるボタンです。

- **［ファイル名をコピー］**：ファイル名をクリップボードにコピー
- **［開く］**：リンクファイルを関連付けられたアプリで開く
- **［リネーム］**：物理ファイル名の変更＋再リンク
- **［削除］**：配置画像をドキュメントから削除
- **［再リンク］／［一括再リンク］**：リンク先の差し替え

### リネーム

**物理ファイルのリネームと再リンクを同時に行います。** ファイルの名前を変えるとリンクが切れる、という手間を1操作にまとめたものです。

拡張子は自動で保持されます。入力欄には拡張子を除いた部分だけが入っていて、うっかり拡張子まで入力しても取り除かれます。同名のファイルが既にある場合は上書き確認が出ます。

同一ファイルをまとめている行なら、その全ての配置がまとめて新しい名前へ再リンクされます。

### 削除

削除対象が**クリップグループ（マスク）の中にある**とき、専用のダイアログが出て2つの削除方法から選びます。

| 選択肢 | 動作 |
| --- | --- |
| 画像のみを削除 | 配置画像だけを消し、クリップグループは残す |
| クリップグループごと削除 | 画像を囲むクリップグループごと消す |

クリップグループの外にある画像なら、通常の確認アラートだけです。

### 再リンク

ボタンのラベルは状況で変わります。同一ファイルをまとめていて、かつその行の配置が複数あるときは **［一括再リンク］** になり、押すと確認のうえ全ての配置を選んだファイルへ差し替えます。配置が1つなら［再リンク］です。

## フォルダー単位の操作

リンクフォルダー一覧で行を選んでから使います。

### フォルダー再リンク

選んだフォルダーを参照している画像を、**別フォルダーの同名ファイルへまとめて差し替えます**。移動先フォルダーに同名ファイルが無いものはスキップされ、件数がステータス行に出ます。

### 拡張子の変更

参照フォルダーと拡張子を選ぶと、**ファイル名の本体部分が同じで拡張子だけ違うファイル**を探して再リンクします。PSD で作業していたものを入稿用に別形式へ差し替える、といった用途です。

選べる拡張子は png / jpg（jpeg）/ psd / tiff（tif）/ webp / avif / gif / ai / pdf です。既定は psd。

`jpg / jpeg` と `tiff` は**2つの拡張子を順に探します**。まず `.jpg`（`.tif`）を探し、無ければ `.jpeg`（`.tiff`）を探す、という順序です。拡張子の大小文字は無視されます。

すでに同じファイルを参照している画像はスキップ扱いになり、対象／成功／スキップ／失敗の件数がステータス行に出ます。

### リンクを収集

**ドキュメントと同じ階層に `Links` フォルダーを作り、リンクファイルをコピーして再リンクします。** 入稿データをまとめるときの操作です。

`Links` フォルダー内に同名ファイルが既にあればコピーはせず、その既存ファイルへ再リンクします。すでに `Links` フォルダー内を参照している画像はスキップです。

ドキュメントが未保存だと、置き場所が決まらないため実行できません。

## その他のボタン

- **［［リンク］パネルを開く］**：Illustrator 標準の［リンク］パネルを開きます
- **［更新］**：ドキュメントから読み込み直します

パレットは開いたままなので、カンバス側で画像を差し替えたり配置し直したりしたら［更新］を押してください。再読み込み後も、選んでいた行は可能な限り選択し直されます。

## 対象

ドキュメント内の PlacedItem（配置されたリンク画像）。埋め込み画像は対象外です。

PPI とカラースペースの取得は PNG / JPEG / PSD のみ対応しています。

## 補足

Illustrator の常駐パレットは、表示している間に DOM への接続を失うという事情があるため、解析・選択・ズーム・再リンク・削除といった DOM を触る処理はすべて BridgeTalk 経由でメインエンジンに委譲しています。

委譲する関数は初回に一度だけメインエンジン側へ常駐させ、以降は呼び出し式だけを送っています。毎回すべての関数本体を送ると、行数が多いぶん目に見えて遅くなるためです。

一方で、ファイルの存在確認・リネーム・コピーといった `File` 操作は常駐エンジンでも動くため、パレット側で完結させています。DOM に触るのは「リンク先の差し替え」の1点だけなので、そこだけを委譲する形です。

ファイル名のクリップボードコピーは、一時的なテキストフレームを作って `app.copy()` する方式です。コピー後は元の選択状態に戻します。

アートボード送りの ◀ ▶ は、フォントのグリフに依存しないよう、ボタン面に三角形をベクターで自前描画しています。UI の明暗設定も見て色を変えています。
