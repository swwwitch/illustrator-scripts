「プレビュー境界」を切り替えたいだけなのに、環境設定ダイアログを開いて、［一般］タブを探して、チェックして、OK。次は「線幅と効果も拡大・縮小」を変えたくて、また開いて、また探して……。

Illustrator の環境設定は項目が多く、しかも**よく触るものほどカテゴリがバラバラ**です。プレビュー境界は［一般］、字形の境界に整列は［テキスト］、ガイドは［ガイド・グリッド］というように、作業中に切り替えたい項目が別々のタブに散っています。

そこで、作業しながら頻繁に切り替える項目だけを1枚のパレットに集めました。常駐パレットなので開きっぱなしにでき、**チェックした瞬間に反映されます**（OKボタンはありません）。

<img alt="" src="" width="50%" />

## 使い方

スクリプトを実行するとパレットが開きます。あとはチェックボックスやドロップダウンを操作するだけで、その時点で環境設定に書き込まれます。

すでにパレットが開いている状態でもう一度実行した場合は、新しく作らずに既存のパレットを前面に出します。

## 操作できる環境設定

### キー増加

カーソルキーで動く量（cursorKeyLength）を設定します。

数値フィールドは直接入力のほか、↑↓キーでも増減できます。

| キー | 動作 |
| --- | --- |
| ↑ / ↓ | ±1（整数に丸め） |
| Shift + ↑ / ↓ | ±10、かつ10の倍数にスナップ |
| Option + ↑ / ↓ | ±0.1 |

0 より小さくはなりません。

右隣の単位ポップアップ（in / mm / pt / pica / cm / Q/H / px）は、**表示単位を変えるだけのものではありません**。選ぶとドキュメントの定規単位（rulerType）そのものが切り替わります。

なお、内部では値を pt で保持しているため、単位を切り替えても実際の移動量は変わりません。表示だけが新しい単位で計算し直されます。

### 変形と整列

| 項目 | 環境設定キー |
| --- | --- |
| プレビュー境界 | includeStrokeInBounds |
| パターンを変形 | transformPatterns |
| 角を拡大・縮小 | policyForPreservingCorners（ON=1 / OFF=2） |
| 線幅と効果も拡大・縮小 | scaleLineWeight |

### 字形の境界に整列

| 項目 | 環境設定キー |
| --- | --- |
| ポイント文字 | EnableActualPointTextSpaceAlign |
| エリア内文字 | EnableActualAreaTextSpaceAlign |

### ガイドと定規

| 項目 | 動作 |
| --- | --- |
| ガイドを表示 | showGuides |
| ガイドをロック | lockGuides |
| ［ビデオ定規］ボタン | `videoruler` メニューコマンドのトグル |

「ビデオ定規」だけはチェックボックスではなくボタンです。環境設定キーではなくメニューコマンドのトグルなので、現在の状態をパレット側で表示できないためです。押すたびに ON / OFF が入れ替わります。

### アートボード名と枠線

| 項目 | 動作 |
| --- | --- |
| アートボード名を表示 | showArtboardLabelOnCanvas |
| カンバスカラーをホワイトに | uiCanvasIsWhite（ON=1 / OFF=0） |
| ハイライトのカラー | 枠線の色を9色のプリセットから選択 |
| ストロークの幅 | 1 / 2 / 3 / 4 から選択 |

ハイライトのカラーは、ライトブルー／サーモンピンク／グリーン／ミディアムブルー／マゼンタ／シアン／ライトグレー／ブラック／イエローの9色です。

現在の設定値がプリセットにぴったり一致しない場合は、RGB の差が最も小さいプリセットを選択状態にして表示します。

アートボード名・枠線カラー・枠線幅は、どれを触っても3つまとめて書き込まれ、そのあと `zoomout` → `zoomin` を実行してキャンバスを再描画します。設定しただけでは画面に反映されないためです。

## 環境設定ダイアログ側で変更したときは

パレットは開きっぱなしなので、その間に環境設定ダイアログや他のパネルで同じ項目を変えると、パレットの表示とズレます。

その場合は**パレットをクリックしてください**。再アクティブになったタイミングで全項目を読み直し、表示を最新の状態に合わせます。

## 補足

Illustrator の常駐パレットは DOM への接続を失うという事情があるため、環境設定への**書き込みはすべて BridgeTalk 経由でメインエンジンに委譲**しています。

一方、**読み出しはパレットのエンジンで直接**行っています。読み出しはエンジンを跨いでも安全なため、非同期の BridgeTalk を経由する必要がありません。

キー増加のフィールドは、↑↓キーを押しっぱなしにすると保存が連打されるため、40ms のデバウンスをかけています。`app.scheduleTask` に渡す文字列はグローバルスコープで評価される都合上、保存関数を `$.global` 経由で公開しています。
