# 水平方向グループ化

[![Direct](https://img.shields.io/badge/Direct%20Link-SmartAutoGroup--yoko.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/group/SmartAutoGroup-yoko.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

横並びに配置したオブジェクトを、行ごとにグループ化したい場面があります。表組みのような一覧、アイコンとラベルの繰り返し、カードを並べたレイアウトなど。

手作業だと「1行目をドラッグで囲んで `Cmd + G`、2行目を囲んで `Cmd + G`」の繰り返しです。行数が増えるほど面倒ですし、選択範囲が隣の行にかかってしまってやり直す、ということも起こります。

ただ、機械的にやろうとすると「同じ行かどうか」の判定が意外に厄介です。**きれいに揃っているつもりでも、実際には数ポイントずれている**ことがよくあるためです。

そこで、**縦のズレをどこまで許容するかをスライダーで決めて**、同じ行とみなせるものをまとめてグループ化するようにしました。

<img alt="" src="" width="50%" />

## 使い方

1. グループ化したいオブジェクトをすべて選択
2. スクリプトを実行
3. ［縦のズレ許容］を調整して［グループ化］をクリック

グループ化が終わると、作成されたグループが選択された状態になります。

## グループ化の条件

### 縦のズレ許容

2つのオブジェクトの**縦のズレ（重なっていない場合はそのすき間）がこの値以内なら、同じ行**とみなします。単位は pt です（初期値 10 pt、範囲 0〜200 pt）。

ポイントは、**横方向の距離はいっさい問わない**ことです。縦さえ揃っていれば、アートボードの左端と右端のように大きく離れていても同じ行として1つのグループになります。

0 にすると、縦が実際に重なっているものだけがまとまります。

判定に使うのは `visibleBounds` です。

### アートボードごと

ONにすると、**別のアートボードにあるオブジェクトは、縦が揃っていても別々のグループ**になります。どのアートボードに属するかは、オブジェクトの中心点がどのアートボードの矩形に含まれるかで判定します。

このチェックボックスの初期状態は、状況に応じて自動で決まります。

| 状況 | 初期状態 |
| --- | --- |
| アートボードが1つだけ | ディム（グレーアウト）してOFF固定 |
| 選択が複数のアートボードにまたがっている | ON |
| 上記以外（アートボードは複数あるが選択は1つのアートボード内） | OFF |

なお、どのアートボードにも中心が乗っていないオブジェクトは、アートボードのまたぎ判定の集計からは除外されます。

## 行のまとめ方

「同じ行」の判定は、隣り合ったもの同士をたどって連結していく方式（DFSによる連結成分の抽出）です。

つまり、A と B が同じ行、B と C が同じ行であれば、**A と C の縦のズレが許容値を超えていても同じグループ**になります。少しずつずれたオブジェクトが並んでいると、数珠つなぎに1つの行としてまとまることがある、ということです。この場合は許容値を小さくして調整してください。

## グループ化されなかったオブジェクト

行として抽出された結果、**メンバーが1つしかない行はグループ化されません**（1つだけをグループにしても意味がないためです）。

そうしたオブジェクトが残った場合は、その個数を示して再実行するかどうかを確認します。［OK］を選ぶとダイアログがもう一度開くので、［縦のズレ許容］を広げて試し直せます。

## 補足

グループ化の実行中はプログレスバーを表示します。行の数が多いときの目安になります。

グループ化には Illustrator 標準の group コマンドを使っていますが、この場合、できたグループは最前面のレイヤーへ移動してしまいます。そのままだとレイヤー構成が崩れるため、グループ化の直後に元のオブジェクトがあったレイヤーへ戻しています。
