# オブジェクトのリサイズ

[![Direct](https://img.shields.io/badge/Direct%20Link-SmartObjectResizer.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/alignment/SmartObjectResizer.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

バラバラのサイズのオブジェクトを「とりあえずそろえたい」という場面があります。アイコンの大きさをそろえる、ロゴを一番大きいものに合わせる、写真を全部アートボードいっぱいに広げる、など。

ただ、**何を基準にそろえるのかは場合によって違います**。幅なのか高さなのか、長辺なのか、面積なのか。しかも「そろえる」だけでは終わらず、たいていはそのあとに整列も必要です。変形パネルで数値を入力し、整列パネルに切り替えてまたクリック、という往復になります。

そこで、リサイズの基準と整列を1枚のダイアログにまとめ、**ラジオボタンを切り替えるたびにその場で結果を確認できる**ようにしました。

<img alt="" src="" width="50%" />

## 使い方

1. サイズをそろえたいオブジェクトを複数選択
2. スクリプトを実行
3. 「リサイズ基準」のラジオボタンをクリック

**ダイアログを開いた直後は、どの基準も選択されていません。** つまり開いただけでは何も起こりません。基準ラジオをクリックした時点で初めてリサイズが実行され、プレビューされます。

基準を切り替えるたびに、いったん元の状態に戻してから計算し直すので、いくつも試して見比べられます。気に入ったら OK、やめるならキャンセルで元に戻ります。

## リサイズ基準

| 基準 | 選択肢 | 動作 |
| --- | --- | --- |
| 最大 | 幅／高さ | 選択中でいちばん大きい幅（高さ）に全部を合わせます |
| 最小 | 幅／高さ | 選択中でいちばん小さい幅（高さ）に全部を合わせます |
| 指定サイズ | 幅／高さ | 入力した数値に全部を合わせます |
| 基準辺 | 長辺／短辺 | 各オブジェクトの長辺（短辺）どうしをそろえます |
| 面積 | 最大／最小 | 面積がいちばん大きい（小さい）ものに面積をそろえます |
| アートボード | 幅／高さ | 選択全体をアートボードの幅（高さ）に合わせ、中央に配置します |
| 裁ち落とし | 幅／高さ | アートボード＋裁ち落とし（片側3mm）に合わせ、中央に配置します |

チェックできる基準は常に1つだけです。どれかをクリックすると他はすべてオフになります。

「指定サイズ」の入力欄はドキュメントの定規単位で表示され、初期値には選択オブジェクトの平均幅が入ります。入力欄では ↑↓キーで数値を増減できます。

| キー | 増減 |
| --- | --- |
| ↑↓ | ±1 |
| Shift + ↑↓ | ±10（10の倍数にスナップ） |
| Option + ↑↓ | ±0.1 |

### 「基準辺」と「最大／最小」の違い

分かりにくいのがこの2つの違いです。

「最大（幅）」は、**幅という決まった辺**を見ます。横長も縦長も関係なく、全オブジェクトの幅がそろいます。

「基準辺（長辺）」は、**そのオブジェクトにとっての長辺**を見ます。横長のものは幅が、縦長のものは高さが基準になるので、結果として「いちばん長い辺の長さ」がそろいます。向きの違うオブジェクトを見た目のボリューム感でそろえたいときはこちらです。

### アートボード／裁ち落としだけは「まとめて」

ほかの基準が**オブジェクトを1つずつ**リサイズするのに対し、「アートボード」と「裁ち落とし」は**選択全体をひとかたまりとして**扱います。選択全体の外接矩形をアートボードの幅（高さ）に合わせて等倍スケールし、オブジェクト間の相対位置もそのまま同じ倍率で拡大縮小されます。最後にアートボード中央へ移動します。

裁ち落としは片側3mm固定で、幅・高さそれぞれ +6mm した寸法に合わせます。オフセットが上下左右対称なので、中心はアートボードの中心と一致します。

## 縦横比保持と片辺のみ

ダイアログ上部のラジオボタンで切り替えます。

- **縦横比保持**：比率を保ったまま拡大縮小します（初期値）
- **片辺のみ**：幅または高さの片方だけを変更します

「片辺のみ」を選ぶと、**基準辺・面積・アートボード・裁ち落としがディム（グレーアウト）します**。長辺／短辺・面積・アートボード合わせは、そもそも縦横比を保つ前提の基準だからです。「片辺のみ」で使えるのは最大・最小・指定サイズの3つです。

切り替えても選択中の基準は保持され、新しいモードで計算し直されます。

## 整列

右側の2つのパネルで指定します。

| 整列（横） | 動作 |
| --- | --- |
| 左 | 左端をいちばん左のものにそろえる |
| 中央 | 中心のX座標を平均位置にそろえる |
| 右 | 右端をいちばん右のものにそろえる |
| 均等 | 上下方向に等間隔で分配 |
| 0間隔 | 上下方向にすき間なく積む |

| 整列（縦） | 動作 |
| --- | --- |
| 上 | 上端をいちばん上のものにそろえる |
| 中央 | 中心のY座標を平均位置にそろえる |
| 下 | 下端をいちばん下のものにそろえる |
| 均等 | 左右方向に等間隔で分配 |
| 0間隔 | 左右方向にすき間なくつなげる |

### パネルの中に「均等」がある理由

パネル名と分配方向が逆に見えるかもしれませんが、これは意図的です。

「整列（横）」パネルの中身は、**縦に積むときのセット**になっています。左端をそろえて（＝横位置の整列）、上下方向に均等配置する（＝縦方向の分配）。この2つを組み合わせて初めて「左揃えの縦積み」ができます。だから同じパネルに入っています。

「整列（縦）」パネルはその逆で、上端をそろえて左右に並べる横並びのセットです。

排他の単位はパネルではなく**動かす座標軸**です。X座標を動かすもの（左／中央／右／「整列（縦）」の均等・0間隔）どうしが排他、Y座標を動かすもの（上／中央／下／「整列（横）」の均等・0間隔）どうしが排他になります。**軸が違えば同時に指定できる**ので、「整列（横）」パネルの中で「左」と「均等」を両方チェックする、という使い方ができます。

なお「均等」「0間隔」は、オブジェクトが2つ以上のときだけ動作します。

整列の状態は、リサイズ基準を切り替えても維持されます。基準を変えて再リサイズしたあと、新しいサイズに対して整列が再適用されます。

## 2つの計測オプション

### テキストをアウトライン境界で計測

テキストの `geometricBounds` は、実際の字形よりも大きめの矩形を返します。行送りやディセンダの余白が含まれるためです。見出しの文字をきっちりそろえたいときにはこれが邪魔になります。

このチェックをオンにすると、**アウトライン化した実際の字形の境界**で計測します。選択にテキストが含まれているときだけ有効になり（自動的にオンになります）、含まれていないときはディムされます。

### プレビュー境界で計測

初期値はオンです。オンのときは線幅や効果を含む見た目の端（`visibleBounds`）、オフのときはパスの端（`geometricBounds`）で計測します。

**このオプションはリサイズだけでなく整列にも効きます。** 計測基準がリサイズと整列でずれていると、太い線のついたオブジェクトが「サイズはそろっているのに端がそろわない」という状態になるためです。

## ボタン

- **［リセット］**：サイズ・位置・整列をすべて元に戻します（ダイアログは開いたまま）。整列チェックもすべてオフになります
- **［キャンセル］**：元に戻して閉じます
- **［OK］**：現在の状態で確定して閉じます

ダイアログの表示位置はセッション内で記憶されます（Illustrator を終了するとリセットされます）。

## 対象

選択中の PageItem 全般。テキストとグループはアウトライン境界の計測時にのみ特別扱いされ、グループ内のテキストは再帰的に集めて計測します。

オブジェクトが選択されていない場合はアラートを出して終了します。

## 補足

「テキストをアウトライン境界で計測」は、元のテキストを壊さないために**複製 → アピアランスの分割 → アウトライン化 → 計測 → 即削除**という手順を踏んでいます。計測のたびにこれを走らせると重いので、結果はキャッシュしています。キャッシュキーにはオブジェクトの現在のバウンズを丸めた値を含めているため、リサイズで形が変わればキーも変わり、古い値を拾いません。

アートボード／裁ち落とし基準では、以前は選択を一時的にグループ化してスケールしていました。現在は一時グループを作らず、クラスタの左上を原点として各アイテムのサイズと相対位置を同じ倍率でスケールする方式です。グループ化・解除を挟まないため、親階層や重ね順が変わってしまうリスクを構造的に回避しています。

幅または高さが0のオブジェクト（線のみ、空テキストなど）は、`resize()` で拡大できず倍率計算が Infinity になるため、100%＝現状維持として扱っています。
