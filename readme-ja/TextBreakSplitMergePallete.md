# テキスト処理

[![Direct](https://img.shields.io/badge/Direct%20Link-TextBreakSplitMergePallete.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/text/TextBreakSplitMergePallete.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

Illustrator のテキストまわりの困りごとは、だいたい細かくて地味です。PDF から流し込んだテキストがバラバラのフレームに散っている、改行と強制改行が混ざっている、和欧間に余計なスペースが入っている、行を1行ずつ入れ替えたい……。

ひとつひとつは10行のスクリプトで片付く話なのですが、そういう小さなスクリプトが増えていくと、**今度はスクリプトを探すほうが面倒**になります。ショートカットも足りません。

そこで、テキスト処理をひとつのパレットに全部まとめました。常駐パレットなので、開いたまま選択を変えながら、必要なボタンを押していけます。

<img alt="" src="" width="50%" />

## 使い方

1. テキストフレーム（またはテキストを含むグループ）を選択
2. スクリプトを実行

ボタンを押すと即座に実行されます。プレビューや［適用］ボタンはありません。取り消したいときは `Cmd + Z` で戻してください。パレットは `Esc` キーでも閉じられます。

すでにパレットが開いているときは、二重に起動せず前面化するだけです。

### ステータス

パレット上部に、現在の選択内容が常に表示されます。

| 項目 | 内容 |
| --- | --- |
| 対象テキスト | 対象になるテキストフレームの総数 |
| ポイント文字 | そのうちポイント文字の数 |
| エリア内文字 | そのうちエリア内文字の数 |
| 改行 | 段落改行の数 |
| 強制改行 | 強制改行の数 |
| タブ | タブの数 |

**実行できない処理のボタンは自動的にディム（グレーアウト）します。** 強制改行が0なら［強制改行→改行］は押せませんし、テキストが1つしかなければ連結系は押せません。何が効く状態なのかがボタンの明暗で分かります。

## 基本タブ

### 改行

［すべて1行に］は少し欲張りなボタンです。複数フレームを縦に連結したうえで、空行を削除し、改行と強制改行をすべて取り除き、さらに行頭行末スペース・和欧間スペース・連続スペースの整理までまとめて実行します。

| ボタン | 動作 |
| --- | --- |
| すべて1行に | 連結＋空行削除＋全改行削除＋スペース整理を一気に実行 |
| 改行のみ | 段落改行を削除（「強制改行を含む」ON で強制改行も対象） |
| 1文字ごとに改行 | 各文字の後ろに改行を挿入 |
| 指定文字で改行 | テキストフィールドの文字の直後で改行 |
| 指定文字数で改行 | 指定した文字数ごとに改行 |
| 強制改行→改行 | 強制改行を段落改行に変換 |
| 改行→強制改行 | 段落改行を強制改行に変換 |

［指定文字で改行］の対象文字は、初期値が `、。，．｡､,.!?！？` です。ここは自由に書き換えられます。すでに改行が続いている箇所には二重に改行を入れません。また、その文字より後ろに文字が残っていない場合（＝末尾の句点）にも改行は入りません。

［指定文字数で改行］は初期値が35文字です。横の「強制改行」チェックボックスを ON にすると、段落改行ではなく強制改行で折り返します。なお、この処理は既存の行ごとに文字数を数え直します。

### 分割

| ボタン | 動作 |
| --- | --- |
| テキストばらし | 改行ごとに別々のテキストフレームへ分割 |
| 〃（書式保持） | 同上。文字書式と位置を保持 |
| タブで分割 | 段落内をタブ位置で分割 |
| 書式を保持（文字で分割） | 1文字ずつ別フレームに分割し、書式を保持 |
| 書式を無視（文字で分割） | 1文字ずつ別フレームに分割し、書式をリセット |

［テキストばらし］と［〃（書式保持）］の違いは、分割の作り方そのものが違います。前者はフレームを複製して中身を差し替え、行送りから計算した位置に並べます。後者は `TextRange.duplicate` で段落ごと複製し、分割前のフレームの下端（縦組みなら左端）の座標を基準に位置を合わせ直します。書式を保ちたいなら後者です。

［タブで分割］は、タブそのものではなく**タブの次の文字の左端 X 座標**を拾って分割後のフレームを置きます。見た目の位置がそのまま維持されるので、表組みっぽいテキストをバラすときに位置が崩れません。

文字単位の分割は、いったん複製をアウトライン化して1文字ずつのバウンディングボックスを取り、そこへ新しいフレームを合わせるという方法をとっています（アウトライン化に失敗した場合は、文字幅を積算するフォールバックに切り替わります）。改行・タブ・半角/全角スペースはフレーム化されません。

［書式を無視］は「完全な初期化」ではなく、**先頭文字のフォントとサイズだけは引き継ぎます**。塗りは黒、ベースラインシフト・回転・トラッキングは0、水平/垂直比率は100%に戻ります。

### 連結

| ボタン | 動作 |
| --- | --- |
| 縦方向に連結 | 上→下の順に連結して1つのテキストへ |
| 横連結（行維持） | 同じ行のテキストを左→右に連結し、行は別フレームのまま |
| 横連結（行統合） | 横に連結したうえで、複数行を1つのテキストに統合 |
| PDFテキスト整形 | 横に連結し、エリア内文字として整形 |

［縦方向に連結］は、各フレームをいったん行単位に分解し、元の行送りを使って仮の Y 位置を再構成してから並べ直します。単純に「選択順」ではなく**見た目の位置順**で連結されるということです。

［横連結（行統合）］と［PDFテキスト整形］は、PDF から流し込んだテキストを想定した処理です。以下のルールで行をつないでいきます。

- 処理前に強制改行を削除する
- 行末が英数字＋ハイフンで、次の行頭も英数字なら、**ハイフンを削除して**つなぐ
- 行末と次の行頭がともに英数字なら、**語間スペースを補って**つなぐ
- 行末が `。！？` で終わる場合（および `.!?` で終わり、かつ全体が ASCII のみではない場合）**だけ**改行を残す

つまり「途中で折り返されているだけの行」はつなぎ、「文が終わっている行」は改行として残す、という判断をしています。行送りは連結後の行の Y 差分から再設定され、禁則は「弱い禁則」になります。

［横連結（行統合）］はポイント文字／エリア内文字を選択内容から自動判定します（混在しているときはエリア内文字を優先）。［PDFテキスト整形］は常にエリア内文字で出力します。

なお、これらは見た目ベースの近似的な連結です。行ごとの細かい書式差や回転までは保持されません。同じ行かどうかの判定は Y 座標の差が 5 以内かどうかで行っています。

## 整形タブ

### タブ・スペース

| ボタン | 動作 |
| --- | --- |
| タブを削除 | タブを削除 |
| タブ→スペース | タブを半角スペースに変換 |
| 行頭行末 | 各行の行頭・行末のスペースを削除 |
| 和欧間 | 和文と欧文の間のスペースを削除 |
| 連続 | 連続したスペースを1つにまとめる |
| まとめて | 行頭行末・和欧間・連続を順に実行 |
| すべて | 半角・全角すべてのスペースを削除 |
| .と,の後 | 半角ピリオド・カンマの直後にスペースを挿入 |

［和欧間］は正確には「**英数字と英数字に挟まれたスペースだけを残し、それ以外のスペースを削除する**」という処理です。英単語の語間は守られますが、和文中のスペースも消えます。

［連続］は半角スペースの連続と全角スペースの連続を別々に処理します。半角の連続は半角1つに、全角の連続は全角1つになります。

［.と,の後］は、直後が空白・数字・ピリオド・カンマのときはスペースを入れません。`3.14` や `1,000` が壊れないようにするためです。

### スペースや記号の変換

「変換前」と「変換後」をそれぞれラジオボタンで選び、［変換］を押します。

| 選択肢 | 対象 |
| --- | --- |
| スペース | 半角スペース・全角スペース |
| アンダースコア | `_` |
| ハイフン | `-` |

前後に同じものを選んでいるときは［変換］がディムになります。

### 変換・リストの除去

- ［全角英数字→半角］：全角の数字・英大文字・英小文字を半角に
- ［半角カナ→全角］：半角カナを全角に（濁点・半濁点の結合、`｡｢｣､･` の変換も含む）
- ［箇条書き］：行頭の `・` `•` `-` `*` を削除
- ［番号リスト］：行頭の番号（`1.` など。全角数字・全角ピリオドも対象）を削除

## 行の編集タブ

左側のリストボックスに、選択中のテキストの各行が並びます。**このリストが機能するのはテキストフレームを1つだけ選択しているときです。**

- ［上へ］［下へ］：選択行を入れ替え
- ［追加］：ダイアログで入力した行を末尾に追加
- ［編集］：選択行を編集（リストのダブルクリックでも可）
- ［削除］：選択行を削除（確認あり）

リストを操作するたびに、その内容がテキストフレームへ書き戻されます。

右側のボタンは行全体への一括処理です。

| ボタン | 動作 |
| --- | --- |
| ソート | 行を文字コード順に並べ替え |
| 文字数順 | 行を文字数の少ない順に並べ替え |
| 反転 | 行の順序を逆に |
| 重複行 | 重複した行を削除（最初に現れたものを残す） |
| 空行 | 空行を削除 |

## 英数字タブ

英字のケース変換です。ボタンの右に**変換結果のプレビュー**が表示されるので、押す前にどうなるか分かります。プレビューは選択中の先頭テキストの内容を使い、40文字を超える場合は省略されます。

| ボタン | 動作 |
| --- | --- |
| すべて大文字に | UPPERCASE |
| すべて小文字に | lowercase |
| 単語の先頭を大文字 | Capitalize Words |
| 文頭のみ大文字 | Sentence case |
| 英語タイトル形式 | Title Case |

［英語タイトル形式］は冠詞・前置詞・接続詞（a, the, of, and, in など）を小文字のまま残す、いわゆる Title Caps の実装です。ただし先頭と末尾に来た場合は大文字になります。`v.` / `vs.` や `AT&T` `Q&A` といった例外にも対応しています。

［単語の先頭を大文字］と［文頭のみ大文字］は、いったん全体を小文字化してから処理します。すべて大文字で書かれたテキストでも意図どおりの結果になります。

## 制御文字ボタン

パレット左下の［制御文字］は、Illustrator の「制御文字を表示」の ON/OFF を切り替えます。ON のときはボタン頭に `✓` が付き、文字色が青くなります。改行と強制改行のどちらが入っているかを目で確認したいときに使います。

**パレットを閉じるとき、このボタンで ON にしていた場合は自動的に OFF に戻ります。**

## 対象

- テキストフレーム
- テキストフレームを含むグループ（再帰的に処理）
- Illustrator が TextRange として返すテキスト選択

## 補足

Illustrator の常駐パレットは、表示している間に DOM への接続を失うという事情があるため、実際のテキスト操作はすべて BridgeTalk 経由でメインエンジンに委譲しています。委譲するコード全体は `encodeURIComponent` で包んで送り、送信先で復元してから実行します。BridgeTalk が本文送信時にバックスラッシュをエスケープしてしまい、`\r` が壊れるのを避けるためです。

ステータス表示の更新にも事情があります。Illustrator 30.x にはタイマー API（`app.scheduleTask` / `setTimeout`）が無いため、選択の変化を定期的に監視できません。そこで「ユーザーがパレットへ操作しに来た瞬間」＝パレットへのフォーカス復帰時とマウスオーバー時に、選択を取り直してステータスに反映しています。このときは件数の集計に必要な関数だけを送る軽量版を使い、毎回すべてを送らないようにしています。

分割・連結の結果は、いったん1つのグループにまとめられます。パレットを閉じるときに、中身が1つだけになったグループは自動的に解除されます。
