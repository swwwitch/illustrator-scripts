# エリア内文字に変換

[![Direct](https://img.shields.io/badge/Direct%20Link-TextWithShapeToAreaType.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/text/TextWithShapeToAreaType.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

ポイント文字で作った見出しやボタンのラベルを、あとから「やっぱりエリア内文字にしておきたい」と思う場面があります。テキスト量が変わっても枠の中で折り返してほしい、縦横中央に収めたい、といった理由です。

ところが手作業でやろうとすると、長方形を描いてエリア内文字にして、元のテキストをコピーして流し込んで、フォントもカーニングも設定し直して……と、**見た目を再現するところで必ず手間が出ます**。パス上文字ならなおさらです。

そこで、ポイント文字・パス上文字・図形＋テキストを、**見た目を保ったままエリア内文字へ変換する**スクリプトにしました。

## 使い方

1. ポイント文字・パス上文字、またはテキストと長方形を選択
2. スクリプトを実行
3. ダイアログでオプションを設定して OK

ポイント文字・パス上文字が選択に含まれていないと、警告して中止します。

## 変換元ごとの挙動

| 変換元 | 動作 |
| --- | --- |
| ポイント文字 | そのまま計測してエリア内文字化 |
| パス上文字 | 内部で一度ポイント文字に分離してから、同じ経路で処理 |
| テキスト＋長方形 | 長方形を複製してエリア内文字のフレームにし、テキストを流し込む |

テキストのみの場合、複数選択していればそれぞれ個別に変換します。一方、**テキスト＋長方形のときは1組（最初のテキスト＋最初の長方形）しか処理しません**。

フレームに使える図形は**長方形のみ**です。それも、軸に平行で、4つの角がすべて直線コーナー（ハンドルを持たない）のものに限ります。長方形以外の閉じたパスだけが図形として選ばれている場合は、警告して中止します。

行揃えとテキストの配置（縦方向）は、変換元によらず常に中央になります。

## 大きさ調整

| 設定 | 動作 |
| --- | --- |
| する | 計測した実寸に、幅・高さの倍率を掛けてフレーム化 |
| しない | 計測した実寸のまま（等倍） |

倍率は %（百分率）で入力します。既定は幅 120%・高さ 160% で、ボタン風に余白を持たせた比率です。「しない」を選ぶと入力欄はディム（グレーアウト）します。

フレームは元テキストの**中心を保ったまま**拡大されるので、位置がずれることはありません。

なお、この倍率が効くのは**テキストのみを選択したとき**です。テキスト＋長方形のときは長方形のサイズがそのままフレームになるため、倍率は関係ありません。

### ボタン背景

「大きさ調整：する」のとき、あるいはテキスト＋長方形のときは、変換後のエリア内文字に**塗りを2枚追加し、長方形のシェイプ効果**をかけてボタン状の背景にします。塗り色は設定せず、追加した塗りは既定のままです。

「大きさ調整：しない」のときは背景を付けません。また、後述の「読み込んだスタイル」を選んだ場合も付きません（スタイル側が見た目を定義するため）。

## グラフィックスタイル

変換後のエリア内文字に適用するスタイルを選びます。

| 選択肢 | 動作 |
| --- | --- |
| 元の見た目 | 変換元テキストの見た目を一時スタイルとして登録し、それを適用 |
| 読み込んだスタイル | AI ファイルから取り込んだスタイルを適用 |

既定は「元の見た目」です。こちらは変換のたびに一時スタイル（`temp_style`）を登録して適用し、処理後に削除するので、グラフィックスタイルパネルに残骸は残りません。

### スタイルの読み込み

［読み込み］ボタンで AI ファイルを選ぶと、そのファイルのグラフィックスタイルを現在の書類に取り込み、**スタイル名からラジオボタンをその場で組み立てます**。選んだファイルのパスとスタイル名は `Folder.userData`（`styles_for_TextWithShapeToAreaType.txt`）に記憶するので、次回以降はそのまま使えます。

［再読み込み］は、記憶しているファイルからスタイルを取り込み直します。ファイルを選び直さずに済むので、別のドキュメントで同じスタイルを使いたいときに便利です。記憶したファイルが無いときはディムします。

取り込みは、対象の AI ファイルを開いてアートボード内をコピーし、一時レイヤー（`// _imported`）へ貼り付けてからレイヤーごと削除する、という手順です。オブジェクトは残らず、スタイルの登録だけが残ります。

「読み込んだスタイル」で選んだスタイルが現在の書類に未登録なら、記憶した AI ファイルから取り込んでから適用します。取り込んだ外部スタイルは、処理後も削除せずに残ります。

## 引き継がれる属性

変換元テキストから、次の属性をエリア内文字へ引き継ぎます。

- 文字の内容
- フォント・サイズ
- 自動カーニング（`kerningMethod`）
- 文字組みアキ量設定（`mojikumi`）

これに加えて、上記のグラフィックスタイルが見た目を担当します。

## 対象

TextFrame（ポイント文字／パス上文字）、PathItem（軸並行・直線コーナーの長方形のみ）

## 補足

フレームのサイズは、`geometricBounds` をそのまま使うのではなく、**複製 → アピアランスを分割 → テキストのアウトライン**という手順で計測した実寸を基準にしています。線やドロップシャドウなどのアピアランスを含めた「見えているサイズ」を取るためです。計測用の複製は計測後に破棄します（計測に失敗した場合は `geometricBounds` にフォールバックします）。

縦方向の中央配置とグラフィックスタイルの登録には、ダイナミックアクションを使っています。エリア内文字のフレーム整列は DOM から不安定にしか触れず、無名グラフィックスタイルの追加はそもそも API がないためです。アクションは実行時に一時ファイルとして書き出して読み込み、処理が終わると自動で破棄するので、アクションパネルには残りません。

パス上文字は、文字ごとの属性（フォント・サイズ・塗り・線・行送り）を退避してから、パスの始点にポイント文字を新規作成し、属性を復元してから元のパス上文字を削除する、という方法で分離しています。
