# 統合文字組みパネル

[![Direct](https://img.shields.io/badge/Direct%20Link-UnifiedTypePanel.jsx-ffcc00.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/jsx/text/UnifiedTypePanel.jsx)

[![Direct](https://img.shields.io/badge/Back%20to%20home-All%20scripts-cccccc.svg)](https://github.com/swwwitch/illustrator-scripts/blob/master/README.md)

---

テキストを1つ整えるだけなのに、文字パネル、段落パネル、文字組みアキ量設定、禁則処理と、いくつものパネルを行ったり来たりすることになります。フォントを変えて、カーニングを直して、行送りを合わせて、最後に文字組みを設定して……という往復自体が、地味に時間を食います。

しかも「本文はこう、見出しはこう」という組み合わせは、だいたいいつも同じです。毎回それを手で組み立て直しているのは、あまり生産的ではありません。

そこで、文字組みに関わる設定を常駐パレット1枚にまとめ、**操作した瞬間に選択中のテキストへ適用される**ようにしました。

<img alt="" src="" width="50%" />

## 使い方

1. 対象のテキストを選択（テキストフレーム、グループ内のテキスト、テキスト編集モードでの範囲選択のいずれでも可）
2. スクリプトを実行

**適用ボタンはありません。** ラジオボタン、スライダー、入力欄を操作した時点で、その場で選択テキストに適用されます。パレットは `Esc` キーで閉じられます。すでにパレットが開いているときに再実行すると、前面化するだけで二重には開きません。

パレットにフォーカスが戻るたびに選択中のテキストの現在値を読み直して UI に反映するので、別のテキストを選び直せば表示も追従します。

## 上部：種別と方針

パレット最上部に、よく使う組み合わせを一発で当てる2つのパネルがあります。

### 種別

| 種別 | 適用される設定 |
| --- | --- |
| 本文 | 和文等幅／行送り 150%／均等配置（最終行左）／ベタ組み／弱い禁則 v2 |
| 見出し | メトリクス／行送り 115%／左揃え／ツメ組み／弱い禁則 v2 |

### 方針

和文・欧文・和欧混在で変わるのは「文字揃え」「行送りの基準」「ハイフネーション」の3つです。これをまとめて切り替えます。

| 方針 | 文字揃え | 行送りの基準 | ハイフネーション |
| --- | --- | --- | --- |
| 和文 | 中央 | 仮想ボディの上 | OFF |
| 欧文 | 欧文ベースライン | 欧文ベースライン | ON |
| 和欧混在 | 欧文ベースライン | 仮想ボディの上 | OFF |

どちらのパネルもクリックすると「文字組み」タブへ自動的に切り替わり、当たった設定が各コントロールに反映されます。

## 「文字組み」タブ

起動時に開くタブです。左に文字属性、右に段落属性が並びます。

### 自動カーニング

| 選択肢 | 内部の値 |
| --- | --- |
| 和文等幅 | METRICSROMANONLY（欧文のみメトリクス） |
| 0 | NOAUTOKERN |
| メトリクス | AUTO |
| オプティカル | OPTICAL |

「メトリクス」を選んだときだけ、プロポーショナルメトリクスが自動的に ON になります。それ以外を選ぶと OFF に戻ります。字間調整パネル最上部のチェックボックスは、この連動状態を表示しています。

### 字間調整

- **文字ツメ**：0〜100%
- **トラッキング**：-100〜500

どちらも入力欄とスライダーの両方から操作できます。スライダーは通常 1 刻み、**Shift を押しながらドラッグすると 10 刻み**にスナップします。入力欄は範囲外の値を入れると自動的にクランプされます。

入力欄では `↑` `↓` キーで増減できます（Shift で 10 単位、Option で 0.1 単位）。

### 文字揃え

「欧文ベースライン」「中央」の2つはラジオボタンとして直接置かれ、残りの4つ（仮想ボディの上/右・下/左、平均字面の上/右・下/左）は「その他」ラジオの右のポップアップから選びます。よく使う2つだけを1クリックで届く位置に出す構成です。

### フォントサイズと行送り

ここがこのパネルでいちばん独特な部分です。**行送りは常に「自動行送り」として設定されます。**

パネルには3つの入力欄があります。

| 欄 | 内容 |
| --- | --- |
| サイズ | フォントサイズ |
| 行送り | 実質の行送り値（サイズ × 行送り%） |
| 行送り% | 段落の「自動行送り量」 |

入力した % は段落の `autoLeadingAmount` に設定され、あわせて自動行送りが ON になります。つまり Illustrator の文字パネル上では行送りは常に「自動」と表示され、**フォントサイズを変えれば行送りも自動的に追従します**。行送りを再適用する必要はありません。

「行送り」欄に直接 pt 値を入れた場合は、フォントサイズから % を逆算して設定します。行送り欄と行送り%欄は、どちらから入れても同じ結果にたどり着けるようになっています。

なお、選択を読み直した直後の「行送り」欄には、サイズ×% の計算値ではなく、**選択テキストの実際の行送り値**が表示されます。

単位はドキュメントの単位設定（`text/units`）に従います。Q/H 環境では、サイズ欄は「Q」、行送り欄は「H」と表示されます。

### 行送りの基準

仮想ボディの上／欧文ベースラインの2択です。行送りの値そのものは変更せず、基準だけを切り替えます。

### 行揃え

左揃え／中央揃え／右揃え／均等配置（最終行左）／両端揃えの5つを、アイコンボタンで選びます。アイコンはスクリプト側で描画しており、Illustrator の UI 明度設定に合わせて明暗が切り替わります（パレットにフォーカスが戻るたびに再判定）。

行揃えを変えるとテキストフレームの見た目の位置がずれることがありますが、このパネルでは適用前後の `visibleBounds` を比べて差分だけフレームを移動させ、**見た目の位置を保ったまま**行揃えを変更します。

### 日本語組版

禁則と文字組みアキ量設定を1枚のパネルにまとめ、それぞれポップアップで選びます。

| 禁則 | 文字組みアキ量設定 |
| --- | --- |
| なし | なし |
| 強い禁則 | 行末約物全角/半角 |
| 弱い禁則 | 約物半角 |
| 弱い禁則 v2 | 行末約物半角 |
| | 行末約物全角 |
| | 約物全角 |
| | ツメ組み |
| | ベタ組み |

**禁則の「なし」は選んでも適用されません。** Illustrator の scripting からは禁則「なし」を設定できないためです（アクション再生が必要）。項目としては並んでいますが、指定しても無視されます。

## 「サイズ」タブ

### フォントサイズ

サイズ・比率（水平／垂直を同じ値でまとめて設定）・実質（サイズ×比率）の3つが並びます。比率が 100% のときは実質＝実サイズなので、「実質」の行は淡色表示になります。

サイズ欄は「文字組み」タブのサイズ欄と同期しており、どちらで編集しても同じ値になります。

### 見かけ←→実サイズ

このボタンは押すたびに向きが変わる相互変換です。**見た目の大きさ（サイズ×比率）を保ったまま**、実サイズと比率の内訳を組み替えます。

- **比率が 100% 以外のとき**：現在の見かけサイズを実サイズに焼き込み、比率を 100% にします（見かけ→実サイズ）
- **比率が 100% のとき**：サイズを変更したあとに押すと、**変更前の見かけサイズ**に戻るような比率を逆算して当てます（実サイズ→見かけ）

後者のために、パネルは「保つべき見かけサイズ」を内部に保持しています。サイズ欄の編集ではこの基準を更新しないので、サイズをいじった後でも元の見た目に戻せる、という仕掛けです。逆に比率を編集したときや、選択を読み直したときは、その時点の見かけが新しい基準になります。

### タイプスケール

基準サイズと倍率から7段のサイズ一覧を生成します。倍率は音程比から選べます。

| 倍率 |
| --- |
| Minor Second 1.067 |
| Major Second 1.125 |
| Minor Third 1.2 |
| Major Third 1.25（既定） |
| Golden Ratio: ½ 1.309 |
| Perfect Fourth 1.333 |
| Augmented Fourth 1.414 |
| Golden Ratio 1.618 |

一覧の段番号は基準サイズを 0 とし、上に2段（-2、-1）、下に4段（1〜4）が並びます。行をクリックすると、そのサイズが選択テキストに適用されます。基準サイズ欄が空のときは、現在のフォントサイズ（取れなければ 12pt 相当）で初期化されます。

## 「定番」タブ

フォント＋自動カーニング・文字ツメ・トラッキングの組み合わせをプリセットとして登録し、クリック1回でまとめて適用します。

| ボタン | 動作 |
| --- | --- |
| 追加 | 現在の選択の設定をプリセットとして保存 |
| 上書き | リストで選択中のプリセットを、現在の選択の設定で更新 |
| 削除 | リストで選択中のプリセットを除去 |
| 書き出し | 登録済みプリセットを JSON ファイルへ（既定はデスクトップ） |

リストは「フォントのみ」と「詳細」（フォント／自動カーニング／文字ツメ／プロポーショナル／トラッキングの5列）で表示を切り替えられます。プリセットは JSON として `Folder.userData` に永続化されます。

リストにはまず PostScript 名が入り、パレット表示後にフォントの実名（和文フォントなら和文名）へ差し替えられます。

## 「使用フォント」タブ

ドキュメントで実際に使われている「フォント＋設定」の組み合わせを走査して一覧にします。

こちらも「フォントのみ」と「詳細」の2表示があり、詳細は使用数／フォント／サイズ／行送り／自動カーニング／文字ツメ／トラッキング／プロポーショナルの8列です。

**クリックの意味が表示モードで変わります。**

- **フォントのみ**：クリックすると、そのフォントだけを選択テキストに適用します
- **詳細**：クリックすると、その行の**組み合わせ全体**（フォント・サイズ・カーニング・ツメ・トラッキングなど）を選択テキストに適用します

［選択したフォントを選択］ボタンは逆方向の操作で、一覧で選んだ条件に一致するテキストをドキュメントから探して選択します。こちらも「フォントのみ」ならフォント一致、「詳細」なら組み合わせ一致で探します。

「フィルター」パネルで走査範囲を絞れます。

| 項目 | 既定 |
| --- | --- |
| ロックされたテキストを含む | ON |
| 非表示のテキストを含む | ON |
| 現在のアートボードに限定 | OFF |

走査は重いので、このタブに切り替えたとき、フィルターを変更したとき、［リストを更新］を押したときだけ実行されます。

## フッター

| ボタン | 動作 |
| --- | --- |
| 制御文字 | 制御文字（改行・スペースなど）の表示／非表示を切り替え |
| リセット | 12pt／比率100%／ツメ0／トラッキング0／メトリクス／欧文ベースライン／左揃え／行送り115%／仮想ボディの上／ツメ組み／弱い禁則 v2 に戻し、アキ（前／後）も自動へ |
| 再読み込み | 選択中のテキストの現在値を読み直して UI に反映 |

## 適用の単位が項目ごとに違います

見落としやすいところですが、「どこまで適用されるか」は項目によって異なります。

| 項目 | 適用範囲 |
| --- | --- |
| 自動カーニング／文字ツメ／フォントサイズ（タイプスケール経由） | 選択が触れた**段落全体** |
| トラッキング／フォントサイズ（サイズ欄）／比率／フォント | **選択範囲**のみ |
| 行揃え／文字組みアキ量設定／禁則／行送り | 段落・フレーム単位 |

たとえばテキスト編集モードで数文字だけ選んで文字ツメを変えると、その文字だけでなく**その段落全体**に当たります。一方トラッキングは選んだ範囲だけに当たります。カーニングやツメは段落内でバラつくと事故になりやすい設定なので、意図的に段落単位に寄せてあります。

## 対象

TextFrame、グループ内のテキスト（再帰的に処理）、テキスト編集モードでの TextRange 選択

## 補足

Illustrator の常駐パレットは、表示している間にドキュメント DOM への接続を失うという事情があるため、実際のオブジェクト操作はすべて BridgeTalk 経由でメインエンジンに委譲しています。委譲するコードは `encodeURIComponent` で包んで送信しています。

行揃えの「左揃え」だけは特別扱いです。Illustrator は scripting からの `Justification.LEFT` の代入を無視することがあるため、フレームを一時的に `resize(200,200)` → `resize(50,50)` して段落属性をリフレッシュさせるという回避策を通しています（位置とマトリクスは復元）。既知バグ専用のフォールバックで、右揃え・中央揃えでは使いません。

禁則が「なし」の段落は、値を読むだけで例外（Error 9563）を投げます。そのため現在値の取得は try で受け、例外なら「なし」として扱っています。

各タブの2つのリスト（フォントのみ／詳細）は、`orientation = "stack"` で同じ位置に重ねてあります。stack グループの高さは子の最大値なので、2つ置いてもパレットが縦に伸びません。加えて、表示前に両方生成しておくことで `onChange` が確実に発火します（表示後に動的生成した listbox はクリックが効かない環境があるため）。
